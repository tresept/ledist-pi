# スクロール後ページ列とフォームリセット

## 目的

スクロールを表示サイクルの終端にせず、完了後に選択済みの通常ページを表示してから次のスクロールへ戻す。WebUIには、未送信の入力だけを初期値へ戻すリセット操作を追加する。

## 表示サイクル

スクロール文字が空の場合の通常ページ列は変更しない。スクロール文字がある場合は、次の無限サイクルを実行する。

1. 日英の行先ページを各3秒表示する。この開始ページでは次駅を併記しない。
2. 右下（種別あり）または下半分全幅（種別なし）でスクロールを開始し、UIで選択された行先・路線名・直通先路線名を3秒ごとに切り替える。スクロールは継続する。
3. スクロール終了後、通常表示として選択済みの次駅、路線名、直通先路線名、種別変更を各3秒表示する。次駅は日英の行先・次駅併記ページを各3秒、路線名と直通先路線名が両方ある場合は既存どおり80x16素材を上下に合成した1ページとする。
4. 先頭へ戻る。選択済み次駅は手順1で既に表示しているため、スクロール後に重複表示しない。

選択されていない要素のページは作らない。スクロール中に種別変更は表示しない。アセット解決・言語フォールバック・停止時に現在フレームを維持する挙動は既存仕様を維持する。

## WebUIリセット

「選択をリセット」ボタンはブラウザ内のフォームだけを初期状態へ戻す。LED表示へのAPI呼出しは行わない。

- アセット選択: `なし`
- スクロール文字: 空
- スクロール速度と輝度: プロフィールから読み込む既定値
- スクロール中切替: 日本語行先・英語行先を有効、路線名・直通先路線名を無効

プロフィール変更時と同じ初期化経路を利用し、初期値の二重定義を避ける。

## テスト

- スクロールありの実行器が、スクロール終了後に通常ページ（路線名、直通先路線名、種別変更）を経て再度スクロール開始すること。
- 次駅を選択した場合、スクロール終了後に日英の行先・次駅併記ページを表示すること。
- リセット操作がフォームの初期値を復元し、表示APIを呼ばないこと。
